# 综合在线题库系统实施计划

## 1. 摘要

在四学科学习平台现有基础上，把仅支持选择题的「题型库」升级为一个覆盖 **选择、填空、计算、证明、画图** 五种题型的独立题库系统。本计划基于当前真实代码状态制定，目标包括：补齐题库数据、修复并增强 KaTeX 公式渲染、扩展 IndexedDB 存储层、重写题库交互模块、补充响应式样式、建设自动化浏览器测试与图像识别脚本、交付部署与公式渲染文档。

### 3.2 公式渲染修复与增强

**文件**：[assets/js/utils.js](file:///d:/Desktop/大学学习网站/assets/js/utils.js)、[assets/css/base.css](file:///d:/Desktop/大学学习网站/assets/css/base.css)

- 重写 `renderMath(root)`：
  1. 若 `window.renderMathInElement` 未就绪，轮询等待最多 10 次（约 1 秒）。
  2. 调用 `renderMathInElement` 时设置 `throwOnError: false`、`errorColor: #ef4444`、`.math-error` 类。
  3. 兜底：扫描 root 内仍未被替换的 `$...$` / `$$...$$` 文本节点，使用 `katex.render` / `katex.renderToString` 逐个渲染；失败节点包裹 `.math-error`。
  4. 对非 Element 节点（如 document）安全跳过。
- CSS 增加：
  ```css
  .katex-display { overflow-x: auto; max-width: 100%; }
  .math-error { border: 1px dashed var(--red); padding: 2px 4px; border-radius: 4px; }
  ```

## 4. 假设与决策

1. **画图题不强制自动判图**：首版通过 canvas 手绘 + 参考图形文字描述 + 用户自评得分实现，避免复杂的图像识别判图。
2. **计算/证明题采用自评模式**：系统展示评分标准，用户按步骤自评，系统记录得分与错题。
3. **题目级收藏/错题使用独立 store**：不与章节学习系统的 `favorites`/`wrongAnswers` 混合，避免 ID 体系冲突。
4. **公式渲染以 KaTeX 为主**：通过 CDN 加载，已能满足数学/经济/数据结构公式；化学 `mhchem` 如需可在 `index.html` 额外加载。
5. **测试脚本默认连接 `http://localhost:8000`**：若端口不同，通过环境变量 `BASE_URL` 覆盖。

## 6. 风险与应对

| 风险 | 应对 |
|---|---|
| 画图题 canvas 在移动端触摸事件异常 | 同时监听 `pointerdown/move/up`，使用 `getBoundingClientRect` 计算坐标 |
| KaTeX 加载过慢导致首次公式未渲染 | `renderMath` 轮询等待，并在路由渲染后再次调用 |
| 新 IndexedDB store 创建后旧数据不兼容 | 提升 DB_VERSION，`onupgradeneeded` 自动创建新 store；旧数据保留 |
| 自动化测试受 Windows Chrome 路径差异影响 | `tests/cdp.js` 支持 `CHROME_PATH` 环境变量覆盖 |
| pixelmatch 对暗色主题/字体微调敏感 | 首次运行生成基线，阈值设为 0.1%；diff 图人工复核 |
