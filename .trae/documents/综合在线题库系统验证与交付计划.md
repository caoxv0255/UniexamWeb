# 综合在线题库系统验证与交付计划

## 1. 摘要

前期已完成题库系统的核心开发：88 道结构化题目（选择/填空/计算/证明/画图）、题库前端模块、公式渲染增强、独立 IndexedDB 存储及 CDP 自动化测试脚本。本计划聚焦 **剩余收尾工作**：补全交付文档、安装依赖、运行三类自动化测试、修复发现的运行时问题，并输出最终验证报告。

## 4. 假设与决策

- **浏览器路径**：测试脚本默认 Windows 路径 `C:\Program Files\Google\Chrome\Application\chrome.exe`，可通过环境变量 `CHROME_PATH` 覆盖。
- **端口**：测试服务器与 Chrome DevTools 使用 8765/9223、8766/9224，避免与开发服务器冲突。
- **视觉基线**：首次运行生成基线，后续运行与之对比；如需更新基线，手动删除 `tests/baselines/bank-content.png`。
- **图像识别**：当前使用像素级对比（pixelmatch）验证显示一致性；OCR 识别（tesseract.js）为可选依赖，本阶段暂不作为门禁，但保留扩展入口。
- **文档位置**：统一放在 `docs/` 目录，与实施计划 `.trae/documents/` 区分。

## 5. 验证步骤

1. `docs/` 目录下存在 `DEPLOY.md`、`FORMULA_RENDERING.md`、`VERIFICATION_REPORT.md`
2. `cd tests && npm install` 成功
3. `node --check` 所有目标 JS 无语法错误
4. `npm run validate` 输出全部题目 LaTeX 校验通过
5. `npm test` 输出端到端测试全部通过
6. `npm run visual` 输出视觉回归通过（或生成基线）
7. 最终验证报告包含上述所有结果摘要
