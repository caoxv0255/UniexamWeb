# 四学科综合学习平台 — 实施计划

## 一、摘要（Summary）

构建一个面向高数二、线性代数、微观经济学、数据结构四学科的综合性在线学习平台。多文件静态站点，原生 ES Modules + KaTeX（CDN）+ IndexedDB，无构建步骤，浏览器直接打开运行。四大核心模块：章节学习、记忆巩固（艾宾浩斯/SM-2）、题型索引、复习总结工具。数据本地存储，支持 JSON 导入/导出。

**数据提取阶段已完成**：4 份 PDF + HTML 参考已提取为 6 个 JS 数据文件（位于 `assets/data/`）。本计划聚焦于**前端实现**（基础架构 + 四模块 + 收尾）。

### 4.3 `assets/js/utils.js`

**职责**：纯辅助函数，无副作用，可被各模块 import。

导出：
- `$(sel, root=document)`、`$$(sel, root=document)`：querySelector 简写。
- `el(tag, props={}, children=[])`：创建元素，`props` 含 `class/style/text/html/dataset/on`。
- `debounce(fn, ms=200)`、`throttle(fn, ms=100)`。
- `fmtDate(ts)`：`YYYY-MM-DD HH:mm`；`fmtDay(ts)`：`YYYY-MM-DD`；`daysBetween(a,b)`：天数差。
- `todayTs()`：今日 0 点时间戳；`addDays(ts, n)`。
- `clamp(v, min, max)`、`pct(part, total)`：百分比 0-100。
- `escapeHtml(s)`、`stripHtml(s)`。
- `hash()`：解析 `location.hash` 为 `["chapter","gaoshu","ch1"]` 数组。
- `navigate(path)`：`location.hash = "#/" + path`。
- `renderMath(root=document)`：调用 `window.renderMathInElement(root, {delimiters:[{left:"$$",right:"$$"},{left:"$",right:"$"}],throwOnError:false})`，try/catch 包裹（KaTeX 未加载时静默）。
- `toast(msg, type="info")`：插入 `.toast` 到 `.toast-container`，3 秒后移除。
- `download(filename, text, mime="application/json")`：Blob 下载。
- `uid(prefix="id")`：`prefix + Date.now().toString(36) + Math.random().toString(36).slice(2,6)`。

### 4.4 `assets/js/store.js`

**职责**：IndexedDB 封装 + localStorage 设置层。

- `openDB()`：打开 `study-platform`（version 1），创建 5 个 objectStore：`progress`（keyPath `id`，索引 `subjectId`）、`reviews`（keyPath `cardId`，索引 `dueDate`、`subjectId`）、`favorites`（keyPath `id`=`subjectId:typeId`）、`wrongAnswers`（keyPath `id`=uid，索引 `subjectId`、`reviewed`）、`notes`（keyPath `id`，索引 `kpId`、`subjectId`、`updatedAt`）。返回 Promise<DB>，缓存单例。
- 通用 CRUD：`dbGet(store, key)`、`dbGetAll(store)`、`dbPut(store, value)`、`dbDelete(store, key)`、`dbClear(store)`、`dbByIndex(store, indexName, value)`。
- 高层 API：
  - 进度：`markLearned(typeId, subjectId)` / `unmarkLearned(typeId)` / `isLearned(typeId)` / `getLearnedIds(subjectId?)` / `getChapterProgress(subjectId, chapterId)` → `{learned, total}` / `getSubjectProgress(subjectId)` → `{learned,total,pct}`。
  - 收藏：`toggleFavorite(typeId, subjectId)` / `isFavorite(typeId)` / `getFavorites(subjectId?)`。
  - 错题：`addWrongAnswer(quizId, subjectId, userAns, correctAns)` / `getWrongAnswers(subjectId?)` / `markWrongReviewed(id)`。
  - 笔记：`saveNote(note)` / `getNote(id)` / `getNotes(subjectId?)` / `getNotesByKp(kpId)` / `deleteNote(id)`。
  - 复习卡片：`getCard(cardId)` / `putCard(card)` / `getAllCards()` / `getDueCards(dateTs)` / `getNewCards(limit)` / `getCardsBySubject(subjectId)`。
- localStorage 层：`getSetting(key, default)` / `setSetting(key, value)`。键名前缀 `sp:`。设置项：`sp:theme`、`sp:dailyNewLimit`（默认 20）、`sp:onboarded`（首次引导已读）。
- `exportAll()`：聚合 5 store + 设置为 `{version:1, exportedAt, progress, reviews, favorites, wrongAnswers, notes, settings}`。
- `importAll(data)`：清空 5 store + 写入设置，逐条 put。

### 4.5 `assets/js/app.js`

**职责**：应用入口、Hash 路由、视图切换、KaTeX 渲染触发。

- 顶部 `import` 四个数据 + 四个模块 + utils + store。
- `const VIEWS = {}`：注册视图渲染函数（首页、学科章页、小节/题型详情、闪卡、题型库、笔记、仪表盘、收藏、错题）。
- `router()`：解析 `location.hash` → `["chapter","gaoshu","ch1-qt1"]`，匹配规则：
  - `#/` 或空 → 首页（dashboard）
  - `#/subject/:subjectId` → 学科章页
  - `#/type/:typeId` → 题型详情
  - `#/memory` → 记忆仪表盘
  - `#/flashcard` → 闪卡复习
  - `#/bank` → 题型索引
  - `#/favorites` → 收藏
  - `#/wrong` → 错题本

## 六、阶段3：模块二·记忆巩固系统

### 6.1 `assets/js/ebbinghaus.js`

SM-2 改进算法，纯函数。

- `const RATINGS = {again:0, hard:1, good:2, easy:3}`。
- `newCard(cardId, subjectId)`：`{cardId, subjectId, easeFactor:2.5, interval:0, repetitions:0, dueDate:todayTs(), lastReview:0, history:[]}`。
- `schedule(card, rating, now=Date.now())`：
  - again：`repetitions=0; interval=0; dueDate=now+10min`（当天重见，用 `todayTs()+1` 简化为明天）。
  - hard：`easeFactor=max(1.3, easeFactor-0.15); interval=max(1, interval*1.2)`。
  - good：`repetitions++; interval = rep==1?1 : rep==2?3 : rep==3?7 : Math.round(interval*easeFactor)`。
  - easy：`repetitions++; easeFactor=min(3.0, easeFactor+0.15); interval = rep==1?2 : rep==2?5 : rep==3?10 : Math.round(interval*easeFactor*1.3)`。
  - 通用：`dueDate = todayTs() + interval*86400000; lastReview=now; history.push({date:now, rating})`。
  - 返回新 card 对象（不 mutate）。
- `isDue(card, dateTs=todayTs())`：`card.dueDate <= dateTs`。
- `masteryLevel(card)`：根据 repetitions 返回 "新学/熟悉/掌握/精通"。
- `forecastNextWeek(cards)`：未来 7 天每日到期数数组，供热力图/柱图。

  - 评分 → `schedule` → `putCard` → 下一张；`.fc-counter` 显示 x/N。
  - 全部完成 → 完成画面 + 返回仪表盘。
- `ensureCardsInit()`：首次进入记忆模块时，为所有题型创建 `newCard` 记录（若该 cardId 不存在）。惰性初始化，避免首次加载慢。
- `renderPractice(stage, typeId?)`：分阶段巩固。
  - 新学（rep=0）：展示完整 steps + 提示。
  - 熟悉（rep=1-2）：填空式（steps 关键词用 `___` 替换，点击揭示）。
  - 巩固（rep≥3）：从 `QUIZZES` 按 subjectId 抽题做选择（复用题型库的自测组件）。
- `drawForgettingCurve(canvas, cards)`：Canvas 绘制。X 轴天数（0-30），Y 轴记忆保留率（1→0）。曲线：`R = e^(-t/S)`，S 由平均 interval 推算。叠加历史复习点散点。
- `drawHeatmap(container, cards)`：生成 7×12 的 cell grid，颜色深浅 = 当日复习次数。tooltip 显示日期+次数。

## 十、假设与决策（Assumptions & Decisions）

1. **题型即学习单元**：实际数据无 sections/knowledgePoints 层级，进度跟踪、记忆卡片均以 `questionType.id` 为粒度。UI 中"知识点"概念映射到题型。
2. **数据结构照搬实际**：不改数据文件，代码适配 `chapters[].questionTypes[]` 结构。
3. **暗色主题**：沿用参考配色，护眼适合长时间学习。
4. **无后端**：云端同步通过 JSON 导入/导出实现。
5. **KaTeX CDN**：`cdn.jsdelivr.net` 加载，auto-render 扫描 `$...$`/`$$...$$`。
6. **ES Modules**：`<script type="module">`，所有 JS 文件用 `import/export`，需通过 `http://` 访问（file:// 下 ES Modules 受 CORS 限制，需用本地服务器如 `python -m http.server` 或 VS Code Live Server 打开）。
7. **惰性初始化记忆卡片**：首次进入记忆模块时为所有题型创建卡片记录，避免首屏加载慢。
8. **共享题型详情渲染**：`renderTypeBody(type)` 抽取为共享函数，章节系统和题型库复用。
9. **Canvas 手绘图表**：遗忘曲线/热力图/环形图用原生 Canvas/SVG，不引图表库。
10. **图标用 emoji/Unicode**：不引图标库。
