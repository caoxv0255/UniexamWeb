# 四学科综合在线学习平台构建计划

## 一、摘要（Summary）

基于 4 份 PDF 题型攻略（共 84 页，120+ 题型）和 1 份 HTML 参考设计，构建一个多文件静态站点，包含四大核心模块：章节学习系统、记忆巩固系统、题型索引系统、复习总结工具。技术栈为原生 ES Modules + KaTeX（CDN）+ IndexedDB，无需构建步骤，浏览器直接打开即可运行。数据本地存储，支持 JSON 导入/导出实现跨设备同步。

### 2.2 HTML 参考与 PDF 内容差异

用户明确指出「html 中的不全，pdf 还是要读取」。对比发现：
- **PDF 比 HTML 丰富**：每个题型含「必背知识（概念表）」「考法与解法（识别特征+解法表）」「标准化做题步骤」「典型例题（含真题年份）」「易错点」，HTML 仅有简化的 knowledge/steps/tip/pitfall。
- **HTML 比 PDF 多覆盖**：HTML 高数二含第8章二重积分、第9章级数（PDF 未覆盖）；HTML 有公式速查表和自测题库。
- **结论**：以 PDF 为权威内容源完整提取，以 HTML 补充 PDF 未覆盖的章节和公式速查/自测题数据。

### 2.3 可用工具

- Python 3.11 + pymupdf（fitz）：PDF 文本提取效果良好，无需 OCR。
- 无构建工具需求：原生 ES Modules 浏览器原生支持，KaTeX 通过 CDN。

### 3.2 技术选型决策

| 关注点 | 选型 | 理由 |
|--------|------|------|
| JS 组织 | 原生 ES Modules（`import/export`） | 无需构建步骤，浏览器原生支持，代码分文件可维护 |
| 路由 | Hash 路由（`#/chapter/gaoshu`） | 静态站点无需服务端配置，刷新不丢失 |
| 数学公式 | KaTeX + auto-render（CDN） | 渲染快、支持 `\frac`、矩阵、求和等；自动扫描 DOM 中的 `$...$`/`$$...$$` |
| 数据存储 | IndexedDB（主）+ localStorage（辅） | 笔记和复习记录数据量大，IndexedDB 容量大；localStorage 存设置 |
| 同步 | JSON 导入/导出 | 无后端，手动备份恢复 |
| 图表 | Canvas 手绘（记忆曲线/进度环形图） | 避免引入 Chart.js 增重，简单图表原生绘制 |
| 富文本 | contenteditable + 工具栏 | 笔记编辑器，支持公式插入（KaTeX 实时预览） |

## 四、四大模块详细设计

### 4.1 模块一：章节学习系统（chapter-system.js + chapters.css）

**功能**：
1. **层级导航**：学科 → 章节 → 小节 → 知识点 → 题型，面包屑导航。
2. **左侧章节树**：可折叠的章节/小节列表，显示每节题型数量和完成度。
3. **知识点展示**：每个小节展示概念表（PDF 提取的表格）、核心知识（含 KaTeX 公式）、关联题型。
4. **进度跟踪**：标记知识点为「已学」，章节进度 = 已学知识点数/总数，环形进度条。学科总进度卡片（首页）。
5. **跨学科关联**：通过 `tags` 字段，点击某知识点时展示其他学科中相同 tag 的知识点（如「极限」关联高数与微经）。

**关键函数**：
- `renderSubjectView(subjectId)`：渲染学科主页（章节树 + 概览）
- `renderSection(subjectId, chapterId, sectionId)`：渲染小节内容
- `toggleKnowledgeLearned(kpId)`：切换知识点学习状态，更新进度
- `getRelatedKnowledge(tags)`：查询跨学科关联知识点

### 4.2 模块二：记忆巩固系统（memory-system.js + ebbinghaus.js + memory.css）

**艾宾浩斯/SM-2 算法**（ebbinghaus.js）：
- 每张「记忆卡片」对应一个题型或知识点，含：`easeFactor`（难度系数，初始 2.5）、`interval`（间隔天数）、`repetitions`（连续答对次数）、`dueDate`（下次复习日期）。
- 评分映射：不会(again)→重置 repetitions=0, interval=0；困难(hard)→easeFactor-=0.15；记得(good)→正常递增；简单(easy)→easeFactor+=0.15。
- 间隔计算：rep=0→1天，rep=1→3天，rep=2→7天，rep≥3→interval×easeFactor。
- 复习排序：按 dueDate 升序，过期优先。

**功能**：
1. **今日复习**：首页仪表盘显示「今日待复习 N 项」「已掌握 M 项」，到期卡片列表。
2. **闪卡复习**：3D 翻转卡片，正面题目+背面答案（含步骤/技巧/易错），4 档评分按钮。评分后自动调度下次复习。
3. **新卡学习**：从未学过的卡片中按章节顺序发放新卡（每日上限可配置，默认 20 张）。
4. **记忆曲线可视化**：Canvas 绘制个人遗忘曲线（按评分历史拟合）、复习热力图（GitHub 风格日历）、各学科掌握度环形图。
5. **分阶段巩固练习**：
   - 新学阶段（rep=0）：展示完整步骤+提示
   - 熟悉阶段（rep=1-2）：填空式练习（隐藏关键步骤）
   - 巩固阶段（rep≥3）：自测选择题（从 quizzes 数据抽取）

**关键函数**：
- `getDueCards()`：查询今日到期卡片
- `rateCard(cardId, rating)`：评分并调度
- `renderMemoryDashboard()`：仪表盘
- `drawForgettingCurve(canvasId)`：绘制遗忘曲线
- `generatePractice(stage)`：按阶段生成练习

### 4.3 模块三：题型索引系统（question-index.js + question-bank.css）

**功能**：
1. **多维度筛选**：学科（多选）、章节、难度（基础/中等/综合）、题型特征（计算/证明/选择/画图/算法设计）、标签。
2. **搜索框**：全文搜索题型标题、知识点、标签。
3. **题型卡片**：展示 badge、标题、难度标签、所属章节、收藏按钮、错题标记。点击展开详情。
4. **详情展开**：核心知识（KaTeX 渲染）、概念表、考法解法表、标准化步骤（编号列表）、典型例题、技巧、易错点。
5. **收藏与错题**：收藏夹视图；错题本视图（来自自测答错的题），可标记「已攻克」。
6. **标准化解题指导**：每个题型固定区段「思路分析 → 关键步骤 → 易错点提示」，符合脑科学分块学习。

**关键函数**：
- `buildIndex()`：从 4 个学科数据构建扁平化题型索引
- `applyFilters(filters)`：应用筛选+搜索
- `renderCard(typeId)`：渲染题型卡片（折叠/展开）
- `toggleFavorite(typeId)`：收藏切换
- `renderFavorites()`/`renderWrongBook()`：收藏/错题视图

### 4.4 模块四：复习总结工具（notes-system.js + notes.css）

**功能**：
1. **知识框架图生成**：选择学科/章节，一键生成树状知识框架图（HTML 嵌套列表 + CSS 缩进连线，非 SVG 依赖），可折叠展开。一键导出为图片需用 html2canvas（CDN）。
2. **富文本笔记编辑器**：
   - 工具栏：加粗/斜体/列表/标题/公式插入/链接/代码块。
   - 公式插入：弹窗输入 LaTeX，实时 KaTeX 预览，插入到光标位置（以 `$$...$$` 标记存储）。
   - 基于 `contenteditable`，输出 HTML。
3. **笔记与知识点关联**：创建笔记时选择关联的知识点/题型，笔记列表可按学科、章节筛选。
4. **导出与打印**：
   - 导出单篇笔记为 Markdown（HTML→MD 转换）。
   - 导出全部笔记为 JSON。
   - 打印当前笔记/知识框架图（`window.print()` + 打印样式）。
5. **章节总结模板**：预置「本章题型总结模板」「错题反思模板」，一键套用。

**关键函数**：
- `renderKnowledgeMap(subjectId, chapterId)`：生成框架图
- `createNote(kpId)`/`editNote(noteId)`：笔记 CRUD
- `insertFormula()`：公式插入弹窗
- `exportNoteMarkdown(noteId)`：导出 Markdown
- `printView(elementId)`：打印

### 4.5 全局功能

- **顶部导航**：Logo + 学科标签页（4 科带色点）+ 工具入口（闪卡/题型库/笔记/仪表盘）+ 数据管理（导入/导出）。
- **响应式**：CSS Grid + 媒体查询，移动端导航折叠为汉堡菜单，章节布局单列。
- **KaTeX 渲染**：每次视图切换后调用 `renderMath(element)` 扫描并渲染。
- **帮助引导**：首次访问显示功能引导浮层（localStorage 标记已读）。

## 五、PDF 内容提取策略（Content Extraction）

由于内容量大（84 页，120+ 题型），采用脚本化提取 + 人工校验：

### 5.1 提取流程

1. **编写提取脚本** `extract_pdfs.py`（执行阶段创建，提取后可删除）：
   - 用 pymupdf 逐页提取文本。
   - 按章节标题（正则 `^第\d+章|^专题\d+|^题型[一二三四...]`）分段。
   - 识别表格（连续多行 `概念|定义` 格式）、步骤列表（`步骤1：`）、例题（`例题：`/`例\d+`）。
   - 输出为 JSON 文件，供后续整理为 JS 数据模块。
2. **人工校验与结构化**：脚本提取的文本需人工校验公式（PDF 提取的数学符号可能丢失上下标），补充为 LaTeX。重点校验：
   - 数学公式（积分、求和、上下标）→ LaTeX。
   - 代码块（数据结构）→ 保持缩进。
   - 表格→转为数组对象。
3. **生成 4 个数据文件**：`assets/data/{gaoshu,xiandai,weijing,datastruct}.js`，每个 `export const xxx = {...}`。

### 5.2 公式处理

PDF 提取的公式常丢失格式（如 `∫ₐᵇ` 提取为 `∫ab`）。处理策略：
- 提取后根据上下文恢复为 LaTeX：`\int_a^b`。
- 数据文件中所有公式用 `$...$`（行内）或 `$$...$$`（块级）包裹。
- KaTeX auto-render 自动渲染。

### 5.3 章节补全

PDF 未覆盖的章节从 HTML 参考补充：
- 高数二：第8章二重积分、第9章级数（HTML 有，PDF 无）→ 从 HTML 提取并升级为完整数据结构。
- 公式速查表、自测题库：从 HTML 参考完整迁移到 `formulas.js`/`quizzes.js`。

## 七、假设与决策（Assumptions & Decisions）

1. **PDF 文本可提取**：已验证 pymupdf 提取效果良好，无需 OCR。数学公式需人工校验恢复为 LaTeX。
2. **无后端**：云端同步通过 JSON 导入/导出实现，不搭建服务器。
3. **内容完整性**：以 PDF 为权威源，HTML 补充 PDF 未覆盖章节（高数二重积分/级数）和公式/题库。
